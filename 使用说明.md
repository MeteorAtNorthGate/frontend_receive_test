这个小工具是我们预计开发的python后端的“迷你版”，
理论上前端推送任意形式的json之后能在这个工具上显示出来，就没有问题了。

（测试截图1、2 是我用fastapi自带的swagger工具推送 测试用json.txt 的截图。）

使用说明

0. 后端启动
	打包exe:
		main.exe 直接运行默认端口是8000，可以带参数，
		例如 main.exe 8001 端口就是8001
	
	源码调试：
		同上，配置好虚拟环境和依赖之后 
		python main.py 或者带参数运行 python main.py xxxx 都是可以的

1. 接口基本信息

	接口地址: http://<服务器IP>:<端口>/receive

	本地测试: http://localhost:8000/receive (或者http://0.0.0.0:8000/receive)

	请求方法: POST

	数据格式: application/json

2. 前端推送代码示例 (JavaScript)

	可以使用 fetch 接口进行快速验证：

	const testData = {
		"test_id": "001",
		"status": "online",
		"data": {
			"temperature": 25.5,
			"device": "Sensor_A"
		}
	};

	fetch('http://localhost:8000/receive', {
		method: 'POST',
		headers: {
			'Content-Type': 'application/json'
		},
		body: JSON.stringify(testData)
	})
	.then(response => response.json())
	.then(res => console.log('推送成功，后端响应:', res))
	.catch(err => console.error('推送失败:', err));



注：本工具仅作为推送验证使用。在实际生产环境下，请确保前端配置的 IP 地址与后端服务所在主机的网络是互通的。
